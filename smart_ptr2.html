第一版的不足

在上篇 的结尾，我们写出一个toy的初级shared_ptr。智能指针不仅仅自动回收资源，而且要和裸指针表现一致，本文将致力于完善它，最终将会呈现给大家一个可以完全替代裸指针的智能指针类。

类型转换

假如有一个Base类和继承自它的一个子类Derived，那么存在从Derived指针到Base指针的隐式转换：

   Derived*  pd = new Derived;
   Base* pb = pd; //  隐式转换
};

我们写出类似的智能指针版本：

    bert::shared_ptr<Derived> pd(new Derived);
    bert::shared_ptr<Base>     pb(pd); // compile error

然后编译，啊噢，好像出了点问题，gcc开始抱怨：

test.cc:13: error: no matching function for call to ‘bert::shared_ptr<Base>::shared_ptr(bert::shared_ptr<Derived>&)’

bert.h:27: note: candidates are: bert::shared_ptr<T>::shared_ptr(const bert::shared_ptr<T>&) [with T = Base]

bert.h:13: note: bert::shared_ptr<T>::shared_ptr(T*) [with T = Base]

给出的错误信息很明确，缺少相应的构造转换函数； 只要你的C++基础及格了，应该很清楚智能指针需要加入一个模板拷贝构造函数：

    template <typename U>
    friend class shared_ptr;
    // shared_ptr<T>和shared_ptr<U>是完全不同的两个类

    template<class U>
    shared_ptr(const shared_ptr<U>& ptr) : ptr_(ptr.ptr_)   {
        count_ = ptr.count_;
        ++ *count_;
    }

别急，还没有结束。如果明知道Base*指向的是一个Derived实例，用裸指针可以这样做：

   Base* pb = new Derived;
   Derived* pd = static_cast<Derived* >(pb); //  显式转换
};

尝试用智能指针这样做，发现编译器报错：

    bert::shared_ptr<Base> pb(new Derived);
    bert::shared_ptr<Derived> pd(pb); // error: invalid conversion from ‘Base* ’ to ‘Derived*’

   // 期待可以这样写：
   // bert::shared_ptr<Derived> pd(static_cast<Derived>(pb));

很显然，Base没有到Derived的自动转换，static_cast也不能应用在智能指针上。没关系，我们造一个智能指针的专属版本：static_pointer_cast. 先让大家回忆一下std::advance，它是一个关于迭代器的算法函数，能够自动识别迭代器类型，进行最高效的移动操作，如果是随机迭代器，算法复杂度O(1)，如果是双向迭代器，算法复杂度O(n)。它是如何做到的呢？查看c++标准头文件，发现了这个秘密:(代码做了一定重写便于阅读)

  template<typename Iterator, typename Distance >
    inline void
    advance(Iterator& it, Distance n)     {
      // 调用辅助函数
      std::__advance(it, n, std::__iterator_category(it));
    }
   //  std::__iterator_category获取迭代器的类型

  template<typename BidirectionalIterator, typename Distance>
    inline void
    __advance(BidirectionalIterator& it, Distance n,
              bidirectional_iterator_tag)    {
      if (n > 0)
        while (n--)
            ++it;   //双向迭代器，O(n)复杂度
      else
        while (n++)
            --it;
    }

  template<typename RandomAccessIterator, typename Distance>
    inline void
    __advance(RandomAccessIterator& it, Distance n,
              random_access_iterator_tag)    {
         it += n; // 随机迭代器，O(1)复杂度
    }

受到标准库的启发，继续完善智能指针：

 struct static_cast_tag
 {     };      

   // for cast
       template <typename U>
       shared_ptr(const shared_ptr<U>& ptr, static_cast_tag ) : ptr_(static_cast<T* >(ptr.ptr_)) {
         count_ = ptr.count_;
         ++ *count_;
    }   

// external  function 
   template <typename U, typename T>
shared_ptr<U>  static_pointer_cast(const shared_ptr<T>&  ptr)   {                   
    return shared_ptr<U>(ptr, static_cast_tag());
}
// 现在这样就合法了：
 bert::shared_ptr<Base> pb(new Derived);
 bert::shared_ptr<Derived> pd(bert::static_pointer_cast<Derived>(pb));
